# 文档编写规范

## 基本原则

1. **结构化优先**：使用表格、列表，避免长段落
2. **状态显式**：所有任务、文件都要标注状态
3. **路由分离**：索引文件只存路由，内容存具体文件

## Markdown 格式规范

### 标题层级

```markdown
# 一级标题 - 文件标题（每个文件只有一个）
## 二级标题 - 主要章节
### 三级标题 - 子章节
```

### 表格规范

表格必须包含表头，用于结构化信息：

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 值1 | 值2 | 值3 |
```

### 状态标注

使用统一的状态符号：

| 符号 | 含义 |
|------|------|
| ✅ | 完成 |
| 🚧 | 进行中 |
| ⏸️ | 暂停 |
| ❌ | 取消 |
| 🟢 | 活跃 |
| 🟡 | 暂挂 |

### Checkbox 使用

任务列表必须使用 checkbox：

```markdown
- [ ] 未完成
- [x] 已完成
```

## 文件命名规范

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 索引文件 | README.md 或 CLAUDE.md | README.md |
| 规范文件 | 小写-连字符.md | document.md |
| 任务文件 | ticket-{id}.md | ticket-001.md |
| 设计文件 | design-{name}.md | feature-a.md |

## 语言规范

- 正文使用中文
- 代码和代码注释使用英文
- 技术术语保留原文（API、JSON、Promise 等）
