# AI Native 核心原则

## 基本原则

| 原理 | 含义 |
|------|------|
| AI 是目标开发者 | 工程结构、文档颗粒度和命名优先服务 Codex 的理解和执行 |
| 有效上下文 | 上下文质量比上下文数量重要；入口文件只保留路由和稳定约定 |
| 规范驱动开发 | 复杂任务先明确需求、设计、技术规范，再进入实现 |
| 代码事实优先 | 代码、测试、配置和运行结果优先于过时文档 |
| 增量维护 | 背景和规范随真实变化更新，不为形式重写 |

## Codex 适配原则

- 使用 `AGENTS.md` 作为 Codex 入口，不使用 `CLAUDE.md`。
- 使用 `.agents/skills/{skill}/SKILL.md` 存放可复用流程，不使用 `.claude/skills/`。
- Codex 不依赖 Claude 的 path frontmatter 自动规则；目录级行为用嵌套 `AGENTS.md` 表达，跨目录规范放在 `.agents/rules/` 并由入口或技能路由。
- 权限和执行风险通过 Codex 沙箱、审批和项目配置控制，不复用 Claude settings。
- 子代理不是默认流程步骤；只有用户明确要求代理协作时才使用。

## 规范约束

- 规范一旦定义，应在相关工作中持续执行。
- 规范变更需要显式更新对应文档。
- 发现规范冲突时，先指出冲突并给出建议处理方式。
- 无法确认的信息标记为“待确认”，不要伪造事实。
