# 文档编写规范

## 基本原则

1. 路由分离：索引文件只放目录、文件链接、状态和关键决策。
2. 结构化优先：表格、清单和短段落优先于长篇解释。
3. 状态显式：任务、阶段、文件和问题都应有清晰状态。
4. 可追溯：需求、设计、实现、测试之间能互相追踪。
5. 可压缩：文档要帮助 Codex 快速选择下一步上下文，而不是制造上下文噪音。

## 状态值

| 状态 | 含义 |
|------|------|
| 待处理 | 已记录，尚未开始 |
| 进行中 | 当前正在推进 |
| 完成 | 已交付并验证或确认 |
| 暂停 | 暂时不推进，但保留上下文 |
| 取消 | 不再推进，保留原因 |
| 待确认 | 需要用户或外部事实确认 |

## 文件命名

| 类型 | 规则 | 示例 |
|------|------|------|
| Codex 入口 | AGENTS.md | AGENTS.md |
| 索引 | AGENTS.md | AGENTS.md |
| 规范 | 小写连字符 | code-style.md |
| 任务 | task.md；多任务用编号 | task-001.md |
| 报告 | 小写连字符 | testing-report.md |

## 路由格式

索引文件只存路由，不存正文。优先使用相对链接：

```markdown
| 文档 | 用途 | 状态 |
|------|------|------|
| [requirements](requirements/requirements.md) | 需求与验收标准 | 完成 |
```

## 语言

- 正文默认中文。
- 代码、标识符、命令、文件路径和接口名称保留英文。
- 注释语言跟随目标工程现有风格；没有既有风格时使用英文注释。
- 不确定事实使用“待确认”，不要写成确定结论。
